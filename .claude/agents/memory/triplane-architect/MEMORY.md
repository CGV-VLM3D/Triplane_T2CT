# Triplane Architect Memory

- **Trial 1 implemented** (`src/models/`, `src/losses/`, `src/data/`, `src/configs/trial1.yaml`): shared-weight transformer encoder (one `TransformerEncoderLayer` called `n_layers` times in a loop), expand+sum+Conv3d decoder. Tests use `emb_dim=64, n_heads=4` to stay CPU-feasible; production config uses 512/8.
- **Expand + contiguous()**: `z_yz.unsqueeze(2).expand(...).contiguous()` is required before Conv3d — `expand()` gives a non-contiguous stride-0 view that Conv3d rejects without it.
- **IdentityAE** (`src/models/identity_ae.py`): no-op baseline; `forward` returns `(mu, {})` — empty triplane dict is intentional, training loop must tolerate it. Carries a single `_dummy` nn.Parameter so `model.parameters()` is never empty and optimizer construction doesn't fail.
- **D3T F_psi encoder** (`src/models/triplane_encoder.py`): replaced axial mean-pool with learnable z_init tokens (one per plane: `z_init_xy/yz/xz`, shape `[1,1,emb_dim]`). Per spatial position, the collapsed-axis tokens are projected to emb_dim, z_init is prepended at pos 0, shared attention runs, and pos 0 is extracted as the aggregate. Permute order for YZ: `(0,3,4,2,1)` to get `[B*W*D, H, C]`; for XZ: `(0,2,4,3,1)` to get `[B*H*D, W, C]`. This is more expressive than mean-pool at the cost of one emb_dim parameter per plane.
- **MAISILatentDataset** (`src/data/maisi_latent_dataset.py`): `stats.json` lives at `<root>/stats.json` (sibling of split dirs, confirmed on disk). `mu.pt` files are float16 on disk — always `.float()` on load. Now supports `load_ct_recon=True`: cache is stored by `measure_upper_bound.py --cache-recon` as fp16 with shape `[1, 1, 480, 480, 256]` (batch dim included); squeeze dim-0 to return `[1, 480, 480, 256]`. Cache path layout: `<ct_recon_root>/<split>/<patient_id>/ct_recon.pt`. Also returns `"sample_id"` as alias for `"patient_id"`. Smoke-check confirmed: 1000 valid samples, ct_recon shape `(1,480,480,256)` float32.
- **src/metrics/** (latent_metrics, image_metrics, upper_bound): monai SSIMMetric win_size must be explicit (default 11 fails when spatial dim < 11; use win_size=7 for tests with small tensors). LPIPS module is module-level lazy cache (`_LPIPS` global) to avoid re-loading VGG weights on every call. `image_psnr_3d` returns +inf on identity (monai PSNRMetric behavior); `latent_psnr` clamps MSE to 1e-10 for finite result instead.
- **Metric unification** (`scripts/train.py`, `src/evaluation/validate.py`): deleted local `_psnr`/`_ssim_3d` from train.py; both files now call `latent_l1`, `latent_psnr`, `latent_cosine_similarity` from `src.metrics`. train.py computes `data_range` once via `compute_latent_data_range` (p99-p1) before the loop; validate.py uses per-batch `max-min` (intentional, higher variance is acceptable at validation frequency). SSIM removed from training metrics entirely — wandb now shows comparable psnr/l1/cosine across train and val phases.
- **_f_psi batch chunking + grad_checkpoint** (`src/models/triplane_encoder.py`): added `chunk_size: int = 2000` and `grad_checkpoint: bool = True` args. N-dim of `[N, seq, dim]` inside `_f_psi` is embarrassingly parallel; split into chunks of `chunk_size`, wrap each chunk's `self.attn(chunk)` call with `torch.utils.checkpoint.checkpoint(..., use_reentrant=False)` when training. Reduces peak from ~16 GiB (unchunked, no-grad) to ~6.5 GiB at chunk=2000. Plan claimed ~1.5 GiB — that was optimistic; real reduction is 2.5× not 10× because CUDA allocator retains freed blocks and `TransformerEncoder` activates all 4 layers before freeing. The test for memory ceiling uses relative assertion (chunked < 50% of unchunked) plus a 10 GiB absolute cap, not a fixed 3 GiB, to stay hardware-agnostic. Propagated through `TriplaneAE.__init__` and `scripts/train.py:build_model` via `getattr(enc, "chunk_size", 2000)`.
- **2026-05-13 — Patchify rewrite (trial2):** Replaced chunked-attention encoder with `nn.Conv3d(4, emb_dim, patch_size, stride=patch_size)` patchify, reducing F_psi sequence batch from 14,400 positions to 900/480/480 at patch_size=4. Removed `chunk_size`/`grad_checkpoint` machinery (kept as silent no-op kwargs for backward compat with old configs). Decoder gets `nn.ConvTranspose3d(hidden, out_channels, patch_size, stride=patch_size)` unpatchify + 2× ResBlock3D before it. Result: 19 ms / 0.205 GiB peak at emb_dim=256, patch_size=4 vs ~6.5 GiB pre-patchify. `chunk_size`/`grad_checkpoint` tests deleted from test_shapes.py (no longer valid). Gotcha: `patch_size` must be identical in encoder and decoder; TriplaneAE default=4. `train.amp: bf16` wraps only the model forward+loss in `torch.autocast(device_type=..., dtype=torch.bfloat16)` — MAISI eval stays fp32.
- **2026-05-14 — trial3 config (out_channels=32):** trial2 plateaued at PSNR≈16.8 dB vs MAISI upper-bound 30.94 dB; root cause was 248× compression (14,880 triplane floats vs 3,686,400 latent floats). `src/configs/trial3.yaml` bumps `encoder.out_channels` and `decoder.in_channels` from 8→32; all other hyperparams identical to trial2. Gotcha: `build_model` in `train.py` passes `enc.out_channels` to `TriplaneAE(out_channels=...)` which then hardwires that same value into the decoder — `dec.in_channels` in the yaml is documented intent only, not read by code. Peak memory at 32 channels: 0.233 GiB (still trivial).
- **2026-05-14 — trial4 config (decoder lift bottleneck):** trial3 showed capacity hypothesis REFUTED (out_channels 8→32 gave no PSNR gain). Latent PSNR 26 dB → image PSNR 17 dB (9 dB drop) points to shallow decoder. trial4 changes decoder only: `hidden: 128` (was 32) and `n_res_blocks: 6` (was 2). Wired `decoder_n_res_blocks` through `TriplaneAE.__init__` and `build_model()` via `getattr(dec, 'n_res_blocks', 2)`. Total params: 8.38M (vs 3.55M for trial2). Peak inference: 0.25 GiB. All 71 tests pass.
- **2026-05-14 — trial4 REVISED (L1→L2 loss swap, decoder reverted):** Decoder changes from earlier trial4 are reverted (back to trial2 baseline: `hidden=32`, no `n_res_blocks`). Only change from trial2 is loss: `l1_weight=0.0, l2_weight=1.0`. `ReconLoss` gained `l2_weight=0.0` param (backward compat preserved). `train.py` logs `loss/l2` in both console print and wandb. Rationale: L1 regresses to median; L2 directly minimizes MSE/PSNR. If PSNR jumps, loss is the bottleneck; if not, architecture is. Smoke test confirmed l1=0.0, l2>0, total=l2, no NaN grads.
- **2026-05-14 — trial5 config (patch_size=2, emb_dim=512):** All trials 2-4 cluster at 15.9–17.0 dB; ceiling is architectural — spatial compression (patch=4) is too aggressive. trial5: `patch_size=2` in both encoder+decoder, `emb_dim=512` (restored from trial1 D3T baseline), `epochs=10` (patch=2 gives 4× more tokens → ~4× compute). Smoke test: 13.6M params, 285 ms fwd+bwd, 25.76 GiB peak, 3.51 it/sec. No model code changes needed — both are already configurable. Gotcha: the task-provided smoke script calls `y = model(x)` without unpacking — model returns `(mu_hat, triplane)` tuple; fix to `y, _ = model(x)` before passing to `loss_fn`.
- **2026-05-17 — Model registry refactor:** Replaced `build_model()` IF chain with `MODEL_REGISTRY` dict in `src/models/__init__.py`. Each model owns `from_config(cls, cfg)`. `TriVQAEConv` in `src/models/trivae_conv.py` is a delegation wrapper (not subclass) to avoid a top-level import of `reference/models/trivae2.py` — that import is deferred into `from_config` since `reference/` is only on `sys.path` at train time. Adding a new model = create file + add 1 line to `__init__.py` + define `from_config`. 65 tests pass.
- **2026-05-17 — TriVAE_D3T (`src/models/trivae_d3t.py`):** D3T-style arch with 3D Swin encoder+decoder. Uses MONAI `PatchEmbed` + `SwinTransformerBlock` / `compute_mask`. Key choices: (1) `SwinStack` caches attention masks at init from `patched_grid_size`, lazily moves to device on first forward (avoids repeated `compute_mask` calls); (2) `FPsiPlane` — independent attn per plane (not shared weights) with per-plane z_init + pos_embed; (3) decoder lifts triplane via `Conv3d(out_ch→emb_dim, 1×1)` then Swin blocks then `ConvTranspose3d`. `build_model()` in `train.py` must handle the flat `cfg.model` layout (no `encoder`/`decoder` sub-keys) — branch was moved to before `enc = cfg.model.encoder` to avoid KeyError. Smoke: 11.45M params, 2.58 GiB peak on A6000, bf16. Gotcha: `SwinTransformerBlock` expects `[B, D, H, W, C]` channel-last format — permute before/after. Window (5,5,4) divides patched dims (30,30,16) for patch_size=4; also divides (60,60,32) for patch_size=2.
