# Triplane Architect Memory

- **Trial 1 implemented** (`src/models/`, `src/losses/`, `src/data/`, `src/configs/trial1.yaml`): shared-weight transformer encoder (one `TransformerEncoderLayer` called `n_layers` times in a loop), expand+sum+Conv3d decoder. Tests use `emb_dim=64, n_heads=4` to stay CPU-feasible; production config uses 512/8.
- **Expand + contiguous()**: `z_yz.unsqueeze(2).expand(...).contiguous()` is required before Conv3d — `expand()` gives a non-contiguous stride-0 view that Conv3d rejects without it.
- **IdentityAE** (`src/models/identity_ae.py`): no-op baseline; `forward` returns `(mu, {})` — empty triplane dict is intentional, training loop must tolerate it. Carries a single `_dummy` nn.Parameter so `model.parameters()` is never empty and optimizer construction doesn't fail.
- **D3T F_psi encoder** (`src/models/triplane_encoder.py`): replaced axial mean-pool with learnable z_init tokens (one per plane: `z_init_xy/yz/xz`, shape `[1,1,emb_dim]`). Per spatial position, the collapsed-axis tokens are projected to emb_dim, z_init is prepended at pos 0, shared attention runs, and pos 0 is extracted as the aggregate. Permute order for YZ: `(0,3,4,2,1)` to get `[B*W*D, H, C]`; for XZ: `(0,2,4,3,1)` to get `[B*H*D, W, C]`. This is more expressive than mean-pool at the cost of one emb_dim parameter per plane.
- **MAISILatentDataset** (`src/data/maisi_latent_dataset.py`): `stats.json` lives at `<root>/stats.json` (sibling of split dirs, confirmed on disk). `mu.pt` files are float16 on disk — always `.float()` on load. Now supports `load_ct_recon=True`: cache is stored by `measure_upper_bound.py --cache-recon` as fp16 with shape `[1, 1, 480, 480, 256]` (batch dim included); squeeze dim-0 to return `[1, 480, 480, 256]`. Cache path layout: `<ct_recon_root>/<split>/<patient_id>/ct_recon.pt`. Also returns `"sample_id"` as alias for `"patient_id"`. Smoke-check confirmed: 1000 valid samples, ct_recon shape `(1,480,480,256)` float32.
- **src/metrics/** (latent_metrics, image_metrics, upper_bound): monai SSIMMetric win_size must be explicit (default 11 fails when spatial dim < 11; use win_size=7 for tests with small tensors). LPIPS module is module-level lazy cache (`_LPIPS` global) to avoid re-loading VGG weights on every call. `image_psnr_3d` returns +inf on identity (monai PSNRMetric behavior); `latent_psnr` clamps MSE to 1e-10 for finite result instead.
- **Metric unification** (`scripts/train.py`, `src/evaluation/validate.py`): deleted local `_psnr`/`_ssim_3d` from train.py; both files now call `latent_l1`, `latent_psnr`, `latent_cosine_similarity` from `src.metrics`. train.py computes `data_range` once via `compute_latent_data_range` (p99-p1) before the loop; validate.py uses per-batch `max-min` (intentional, higher variance is acceptable at validation frequency). SSIM removed from training metrics entirely — wandb now shows comparable psnr/l1/cosine across train and val phases.
